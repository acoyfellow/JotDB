Cloudflare Durable Objects

Durable state without migrations.

JotDB is a tiny database for stateful Cloudflare apps. Use one Durable Object per user, tenant, room, session, or actor — with schema validation built in.

View on GitHubSee examples

User preferences

One object per user, no table required.

const db = env.JOTDB.getByName(`user:${userId}`)
await db.setAll({ theme: "dark" })

Feature flags

One object per tenant, typed and isolated.

const flags = env.JOTDB.getByName(`tenant:${id}`)
await flags.set("aiSearch", true)

Chat history

One object per room, append messages directly.

const room = env.JOTDB.getByName(`room:${id}`)
await room.push({ body: "hello" })

Why JotDB instead of D1?

Use JotDB when

  • Data belongs to one user, tenant, room, or session
  • You want no SQL or migration files
  • You want schema validation at the edge
  • You need actor-style isolation and consistency

Use D1 when

  • You need joins or ad hoc queries
  • You need reporting or analytics
  • Your data is relational
  • You need cross-entity transactions

Cloudflare Durable Objects

JotDB

A tiny durable database for stateful Cloudflare apps. No SQL. No migrations. No ORM. Just per-entity state with schema validation.

User preferences

One Durable Object per user.

const db = env.JOTDB.getByName(`user:${userId}`) await db.setAll({ theme: "dark" })

Feature flags

One Durable Object per tenant.

const flags = env.JOTDB.getByName(`tenant:${id}`) await flags.set("aiSearch", true)

Chat history

One Durable Object per room.

const room = env.JOTDB.getByName(`room:${id}`) await room.push({ body: "hello" })

Why not D1?

Use D1 for joins, analytics, and relational queries. Use JotDB when your data naturally belongs to one user, tenant, room, session, or actor and you want schema validation without migration files.
